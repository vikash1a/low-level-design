### SOLID
 - Single responsiblity principle
   - class should have a single responsibility 
 - Open Closed principle
   - open for extension, closed for modification 
 - Liskov substiution principle
   - Sub Class should be substituted by its base class
 - Interface segeration 
   - No client should be forced to depend on methods it does not use.
   - prefer mutiple interfaces over single generic interfaces
 - Dependency Inversion
   -  High-level modules should not depend on low-level modules 
   - Inject dependency by interface/abstraction instead of implementation

### Other
- KISS - keep it simple stupid
- YAGNI - You are not going to need it
